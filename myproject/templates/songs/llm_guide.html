STEP 2 ノート重要度スコアリング

概要

コクヨノートをOCRしたテキストから、赤字・太字・下線などの視覚的特徴を使って重要ワードをスコアリングします。

視覚マーカー形式

OCRテキストに以下のマーカーを付けて精度を上げます:

マーカー重み例
[red]...[/red]10.0赤字 (最重要)
[box]...[/box]9.0枠囲み
[bold]...[/bold]8.0太字
【...】8.0括弧強調
[highlight]...[/highlight]7.0蛍光ペン
[underline]...[/underline]6.0下線
[star]...[/star]5.0★マーク
コマンドラインで実行
# ルールベースのみ (高速、LLM不要)
python -m note_importance.scorer \
  --input note.txt --mode rule

# ハイブリッド (ルール + LLMリファイン)
python -m note_importance.scorer \
  --input note.txt --mode hybrid \
  --model Qwen/Qwen2.5-1.5B-Instruct

# ディレクトリ一括処理
python -m note_importance.scorer \
  --input-dir ./ocr_output/ \
  --output results.jsonl
モード比較
モード精度速度GPU必要用途
rule ⭐⭐⭐ ⭐⭐⭐⭐⭐ (即時) 不要 初期チェック・大量処理
hybrid ⭐⭐⭐⭐⭐ ⭐⭐⭐ (数秒/件) 4GB+ VRAM 最終スコアリング

STEP 3 歌詞生成LLM学習 (本能寺の変スタイル)

スタイル参考: エグスプロージョン「本能寺の変」
  • 学習内容をリズミカルに歌詞化
  • 韻を踏む (ラップ/ヒップホップ的要素OK)
  • ダジャレ・語呂合わせを積極使用
  • 繰り返しサビ + 合いの手で覚えやすく
  • 「覚えよう」等のメタ的表現は使わない
歌詞データセット作成
# テンプレートサンプル生成 (API不要)
python build_lyrics_dataset.py --template --output data/lyrics_samples.json

# 重要度スコアリング結果 → 歌詞データのシード作成
python build_lyrics_dataset.py \
  --input data/importance_results.jsonl \
  --output data/lyrics_seeds.json

# Gemini APIで大量生成
python build_lyrics_dataset.py \
  --input data/lyrics_seeds.json \
  --use-gemini \
  --output data/lyrics_training.json
モデル学習
# 自宅PC (4060 Ti 16GB) - シングルGPU
python train.py \
  --data_path data/lyrics_training.json \
  --model_name Qwen/Qwen2.5-7B-Instruct \
  --epochs 5

# 学校PC (4080 x2) - マルチGPU
accelerate launch train.py \
  --data_path data/lyrics_training.json \
  --model_name Qwen/Qwen2.5-14B-Instruct \
  --epochs 5

STEP 4 学校GPUにSSH接続

初回セットアップ (学校PC側)
# 1. SSH公開鍵を登録
ssh-copy-id student@192.168.x.x

# 2. 学校PCにプロジェクトを配置
scp -r training/ student@192.168.x.x:~/utamemo-training/

# 3. 学校PCでvenv作成 & 依存インストール
ssh student@192.168.x.x
cd ~/utamemo-training
python3 -m venv venv
source venv/bin/activate
pip install -r requirements_training.txt
自宅PCからSSHトンネル (推論サーバーアクセス)
# 学校PCのserve.pyに自宅からアクセスする
ssh -N -L 8000:localhost:8000 student@192.168.x.x

# → localhost:8000 で学校PCの推論サーバーにアクセス可能
Gradio UIでの操作
  1. SSH接続タブでホスト・ユーザー・鍵パスを入力して保存
  2. 接続テストボタンでGPU情報が表示されればOK
  3. コード同期でtraining/を学校PCに転送
  4. 学習管理タブで学習開始・ログ監視・モデルダウンロード

推奨モデル一覧

モデル VRAM目安 4060Ti 16GB 4080 x2 用途
Qwen/Qwen2.5-1.5B-Instruct ~4GB ✅ 余裕 ✅ テスト・デバッグ
Qwen/Qwen2.5-7B-Instruct ~8GB ✅ OK ✅ 歌詞生成・重要度判定
meta-llama/Meta-Llama-3-8B-Instruct ~10GB ✅ OK ✅ 歌詞生成 (高品質)
Qwen/Qwen2.5-14B-Instruct ~16GB ⚠️ ギリギリ ✅ 高品質生成
google/gemma-2-9b-it ~12GB ✅ OK ✅ バランス型

トラブルシューティング

  • --batch_size 1 に下げる
  • --gradient_accumulation 8 に上げる (同等の効果)
  • より小さいモデルに変更: Qwen2.5-7B → Qwen2.5-1.5B
  • --lora_rank 16 に下げる

  • 学校WiFiに接続しているか確認
  • 学校PCが起動しているか確認
  • IPアドレスが変わっていないか確認: ssh student@{IP}
  • ファイアウォール設定の確認
