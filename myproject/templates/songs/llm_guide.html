クイックスタート

以下をコピペすればすぐに動きます。

# 1. リポジトリをクローン
git clone https://github.com/Yulkjh/utamemo-app.git
cd utamemo-app/training

# 2. 依存インストール (初回のみ)
pip install -r requirements_training.txt

# 3. トレーニングエージェント起動
#    ダッシュボードから「開始」で自動ループ (データ生成→学習→繰り返し)
python training_agent.py \
  --api_key YOUR_API_KEY \
  --report_url https://utamemo.com/api/training/update/ \
  --gemini_key YOUR_GEMINI_KEY \
  --gen_count 5

起動後、上の「LLM Training Monitor」から「トレーニング開始」を押すと自動でデータ生成→学習が無限ループします。「停止」で中断。

# 1. SSHで接続
ssh user@your-gpu-server

# 2. リポジトリをクローン & セットアップ
git clone https://github.com/Yulkjh/utamemo-app.git
cd utamemo-app/training
pip install -r requirements_training.txt

# 3. tmux/screenでバックグラウンド実行 (SSH切断しても継続)
tmux new -s training
python training_agent.py \
  --api_key YOUR_API_KEY \
  --report_url https://utamemo.com/api/training/update/ \
  --gemini_key YOUR_GEMINI_KEY \
  --gen_count 5

# tmux デタッチ: Ctrl+B → D
# tmux 再接続: tmux attach -t training

tmux内で起動すればSSH切断後も学習が継続します。ダッシュボードからリモート監視・停止が可能。

コクヨノートの写真 → LLM学習までの全体フロー

コクヨノートの写真がどうやってLLMの学習データになるか、ステップごとに説明します。

STEP A
ノートを撮影
STEP B
OCR テキスト化
STEP C
重要度スコア
STEP D
歌詞データ化
STEP E
LLM学習

  • ノートをできるだけ平らにして撮影 (歪みがあるとOCR精度が落ちる)
  • 明るい場所で影が入らないように
  • 赤字・蛍光ペン・枠囲みがハッキリ写るようにする (これが重要度判定の元になる)
  • 1ページずつ撮影。複数ページもOK
ノートに赤字や太字で書いた部分がそのまま「重要ワード」として抽出されます。普段のノートの書き方がそのまま学習データの品質に直結します。

Google Gemini (Vision) またはローカルの Qwen2.5-VL が画像を読み取り、テキストに変換します。このとき色・太さ・装飾も認識してマーカーを付けます。

アプリ側: Gemini API使用 / 開発側 (Gradio UI): Qwen2.5-VL ローカル実行 (APIキー不要)
入力 (ノート写真)
織田信長は1534年に生まれた。
桶狭間の戦いで今川義元を破り...
天下布武を掲げた。
出力 (OCRテキスト)
[red]織田信長[/red]は[underline]1534年[/underline]に生まれた。
[highlight]桶狭間の戦い[/highlight]で今川義元を破り...
[box]天下布武[/box]を掲げた。

UTAMEMOアプリで画像をアップロードすると自動的にこの処理が行われます。手動でテキストファイルを用意してマーカーを付けてもOKです。

OCRテキストのマーカーを解析して、各ワードに重要度スコアを付けます。

ワード検出マーカースコア
天下布武[box] 枠囲み10.0
桶狭間の戦い[highlight] 蛍光ペン9.0
織田信長[red] 赤字8.0
1534年[underline] 下線6.0
今川義元なし (出現頻度で検出)2.0
 
# 実行コマンド
python -m note_importance.scorer --input ocr_output.txt --mode rule
# → results.jsonl にスコア付きデータが出力される

スコアの高いワードをサビで繰り返す「本能寺の変」スタイルの歌詞データを作成します。

入力 (スコア結果)
織田信長 → 10.0
天下布武 → 9.0
桶狭間の戦い → 7.0
出力 (歌詞データ)
{
  "instruction": "織田信長について歌詞を作って",
  "output": "信長 信長 織田信長\n天下布武を掲げたぜ\n桶狭間で奇襲だ..."
}
# スコア結果 → 歌詞データ
python build_lyrics_dataset.py \
  --input data/importance_results.jsonl \
  --output data/lyrics_training.json

歌詞データで既存のLLM (Qwen等) をQLoRAファインチューニングします。元のモデルの知識を保ちつつ、「エグスプロージョン風」の歌詞生成を学習します。

# 学習実行
python train.py \
  --data_path data/lyrics_training.json \
  --model_name Qwen/Qwen2.5-7B-Instruct \
  --epochs 5

学習後のモデルは serve.py で推論サーバーとして起動し、UTAMEMOアプリから呼び出せます。

まとめ: ノートに赤字や蛍光ペンで書いた部分が自動的に「重要ワード」になり、それを中心にした歌詞学習データが作られます。
つまり 普段のノートの取り方 → そのまま LLMの学習データの品質 に直結します。

2 ノート重要度スコアリング

OCRテキストに以下のマーカーを付けると精度が上がります。
重み = 周囲とのコントラスト（視覚的な際立ち度）で決定。枠や蛍光ペンなど「面積で目立つ」装飾ほど高スコア。

マーカー重み意味
[box]...[/box]10.0枠囲み
[highlight]...[/highlight]9.0蛍光ペン
[red]...[/red]8.0赤字
[bold]...[/bold]7.0太字
[underline]...[/underline]6.0下線
 
# ルールベースのみ (GPU不要・高速)
python -m note_importance.scorer \
  --input note.txt --mode rule

# ルール + LLMリファイン
python -m note_importance.scorer \
  --input note.txt --mode hybrid \
  --model Qwen/Qwen2.5-1.5B-Instruct

3 歌詞生成LLM (本能寺の変スタイル)

学習内容をリズミカルに歌詞化。韻・ダジャレ・繰り返しサビで覚えやすく。

# --- Track B: lyrics_generation パッケージ --- #

# テンプレートから学習データ生成 (GPU不要)
python -m lyrics_generation.dataset_builder template --output data/lyrics_dataset.json

# 重要度結果からシード生成 → Gemini APIで歌詞生成
python -m lyrics_generation.dataset_builder seeds --input data/importance_dataset.jsonl --output data/lyrics_seeds.json
python -m lyrics_generation.dataset_builder generate --input data/lyrics_seeds.json --output data/lyrics_dataset.json --use-gemini

# モデル学習 (自宅PC)
python -m lyrics_generation.train_lyrics train \
  --data data/lyrics_dataset.json \
  --model Qwen/Qwen2.5-7B-Instruct \
  --epochs 5

# 学習後の品質評価
python -m lyrics_generation.train_lyrics evaluate \
  --model-path output/lyrics-lora/

# 学校PC (マルチGPU)
accelerate launch -m lyrics_generation.train_lyrics train \
  --data data/lyrics_dataset.json \
  --model Qwen/Qwen2.5-14B-Instruct \
  --epochs 5

トラブルシューティング

  • --batch_size 1 に下げる
  • --gradient_accumulation 8 に上げる
  • より小さいモデルに変更
  • --lora_rank 16 に下げる
